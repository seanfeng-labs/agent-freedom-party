03 · 核心流程 & 令牌调度
发言全流程
1
用户开题
topic_open → session.state = active
2
Agent 举手
raise_hand { intent, priority, reason_code }
3
调度器评分放行
hand_accepted { token_id, ttl_sec }
4
持令牌发言
speak → 验证 token_id · holder · 未过期
5
释放令牌
yield_token 或 TTL 超时自动回收
6
收束总结
close_request → final_summary
优先队列评分公式
score= priority_weight +wait_bonus +mention_bonus −dominance_penalty
项目 值 备注
priority high +100 紧急风险
priority medium +60 默认
priority low +30 补充
wait_bonus +1/s 上限 30
user 点名 +40 retarget
Agent @ 提及 +15 reply_to
dominance_penalty −20~60 60s 内发言多
04 · 关键事件类型
topic_open
用户开题，携带 topic 字段，触发 idle→active
raise_hand
Agent 请求发言，附 intent / priority / reason_code
hand_accepted
调度器放行，分配 token_id 与 ttl_sec
hand_rejected
拒绝举手；reason: 重复 / 离题 / 限流 / closing
speak
正式发言，验证 token_id · holder · 未过期
yield_token
主动结束发言，立即释放令牌进入下一排队
interrupt_request
请求打断，用于纠错或高风险场景
user_message
用户插话，自动最高优先级，无需举手
retarget
点名指定某 Agent 发言，mention_bonus +40
close_request
发起收束，active→closing，仅 Host 可发言
final_summary
主持输出结论 / 分歧 / 行动项，触发 closing→closed
interrupt_granted
打断获准，抢占令牌，当前持有者被强制 yield
